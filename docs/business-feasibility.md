## 一、執行摘要

### 一句話定位

> **幫埔里茭白筍農民「秒看」全台 16 個市場哪裡最好賣的 LINE 警報工具。**

### 核心發現

| 面向 | 結論 |
|------|------|
| 市場缺口 | **真實存在** — 台灣目前零個工具提供「跨市場價差比較 + 主動警報」 |
| 目標市場規模 | 茭白筍年產值 **16.6～20.2 億元**，埔里占 85%～90% |
| 目標用戶數 | 埔里茭白筍農戶約 **1,500～2,000 戶** |
| 付費意願 | **低但可撬動** — 農民不為「軟體」付費，但會為「多賺的錢」付費 |
| 最佳商業模式 | **Freemium（免費增值）+ 農會 B2B 授權**，而非純訂閱制 |
| 最大風險 | 農民對數位工具訂閱的接受度不確定，需實際驗證 |
| 商業化時程 | 最快 **6～12 個月** 可產生首筆收入 |

## 二、市場機會

### 2.1 茭白筍市場規模

| 指標 | 數據 |
|------|------|
| 全台栽培面積 | 1,835～2,013 公頃 |
| 全台年產量 | 約 4.5 萬公噸 |
| 全台年總產值 | **16.6～20.2 億元**（近十年翻倍成長） |
| 埔里栽培面積 | 約 1,700～1,800 公頃 |
| 埔里占全台比例 | 面積 85%、產量 90% |
| 埔里年產值 | 超過 **10 億元** |
| 批發均價 | 帶殼 ≈ NT$70/kg、去殼 ≈ NT$145/kg |
| 每公頃年產量 | 15～20 公噸 |
| 農戶年毛收入 | NT$100～300 萬/戶 |

> [!important] 市場集中度極高
> 85% 的茭白筍來自埔里，這意味著只要打進埔里，就等於覆蓋了整個產業。地理集中度是獨立開發者的最大優勢 — 不需要全台跑業務。

### 2.2 核心痛點：資訊不對稱

台灣農產品運銷的結構性問題：

```
農民（資訊劣勢）
  ↓ 賤價求售
盤商（資訊優勢）→ 掌握各市場即時行情
  ↓
批發市場（16 個，價差 20～40%）
```

**具體場景**：同一天同一品質的茭白筍，台北市場可能比台中貴 NT$15～30/kg。一車 2,000 kg 的貨，選對市場就多賺 **NT$30,000～60,000**。但農民不知道。

**現有工具都不解決這個問題**：

- 農糧署網站 → 只能「查」，不會主動「告訴你」
- 行情 APP → 通用型，不針對特定作物
- 盤商口頭報價 → 有利益衝突

### 2.3 你的系統正好填補的缺口

| 功能 | 農糧署網站 | 行情 APP | 盤商 | 你的系統 |
|------|-----------|----------|------|---------|
| 跨市場價差比較 | ❌ | ❌ | 口頭（不可信） | ✅ |
| 主動警報推播 | ❌ | ❌ | 看心情 | ✅（LINE Bot） |
| 茭白筍專用分析 | ❌ | ❌ | ❌ | ✅ |
| 歷史趨勢圖 | 部分 | 部分 | ❌ | ✅ |
| 最佳市場建議 | ❌ | ❌ | ❌ | ✅ |

## 四、商業模式選項

### 4.1 三種可行模式比較

#### 模式 A：純 B2C 訂閱制

```
農民 → 直接付費訂閱 LINE Bot 警報
```

| 優點 | 缺點 |
|------|------|
| 直接、簡單 | 農民付費意願低 |
| 不依賴中間人 | 客戶獲取成本高 |
| 用戶直接反饋 | 需逐個說服 |

**收入預估**：100 農戶 × NT$200/月 = NT$20,000/月 = **NT$24 萬/年**

#### 模式 B：B2B2F（透過農會）⭐ 推薦

```
你 → 授權給農會 → 農會推廣給農民（免費或內含在會費）
```

| 優點 | 缺點 |
|------|------|
| 一次成交覆蓋大量農民 | 農會決策慢（3～6 個月） |
| 農民信任農會背書 | 需要政府採購流程 |
| 可申請政府補助 | 客製化需求多 |

**收入預估**：3 個農會 × NT$60,000/年 = **NT$18 萬/年**

#### 模式 C：Freemium 混合（推薦的起步模式）⭐⭐

```
免費 Web 儀表板（引流）
     ↓
免費 LINE Bot 基本警報（黏著）
     ↓
付費 LINE Bot 進階功能（轉化）
     ↓
農會白標授權（規模化）
```

| 階段 | 內容 | 定價 |
|------|------|------|
| 免費層 | Web 儀表板 + 每日 1 則 LINE 行情摘要 | NT$0 |
| 農民版 | 即時價差警報 + 30 天歷史 + 最佳市場建議 | NT$99～299/月 |
| 農會版 | 多農民帳號 + 整合報表 + 品牌白標 | NT$3,000～10,000/月 |

> [!important] 推薦策略
> **先走免費 → 收集口碑 → 再談付費**。農民看到「今天多賺 NT$15,000」的具體數字後，NT$299/月 就不是問題。

### 4.2 政府補助機會

| 補助計畫 | 補助比率 | 門檻 |
|----------|----------|------|
| 智慧農業業界參與補助 | 50% | 需法人資格 |
| 雲世代農業數位轉型 | 依案 | 需法人或農民團體 |
| 農業科技專案計畫 | 依案 | 學術或法人 |

> [!warning] 現實挑戰
> 大多數補助要求**公司法人**資格，個人開發者難以直接申請。可考慮：
> 1. 成立一人公司（行號或有限公司）
> 2. 與農會共同申請（農會為申請主體，你為技術合作方）

## 五、定價策略

### 5.1 農民付費心理學

台灣農民對「軟體」的付費心理障礙：

| 障礙 | 解法 |
|------|------|
| 「軟體」聽起來不值錢 | 改稱「**行情通報服務費**」，比照農會服務 |
| 不習慣月付 | 提供**年繳**（打 7 折），對齊採收後付款 |
| 不確定有沒有用 | **前 90 天免費**，不需綁卡 |
| 不信任陌生開發者 | 透過**農會背書**，或找 5 個農民代言 |

### 5.2 錨點定價法

> 「行情通報服務每月 NT$299，等於 **2 公斤茭白筍** 的收購價。只要因此多賣到更好的市場一次，一年的費用就賺回來了。」

具體 ROI 計算範例：

```
一車茭白筍 = 2,000 kg
選對市場多賺 = NT$15/kg × 2,000 kg = NT$30,000
年訂閱費 = NT$299 × 12 = NT$3,588
ROI = 30,000 / 3,588 = 836%
```

### 5.3 季節性定價

茭白筍一年兩收（春 4～6 月、秋 8～11 月），建議：

- **產季方案**：NT$499/季（只在產季收費）
- **年費方案**：NT$1,999/年（月均 NT$167，強調便宜）
- **農會團購**：10 人以上 5 折

## 六、通路與獲客

### 6.1 LINE Bot 作為主要通路

> [!warning] 重大更新
> **LINE Notify 已於 2025 年 3 月 31 日終止服務！** todo.md 中的「Line Notify 價差提醒」必須改為 **LINE Messaging API（LINE Bot）**。

#### LINE Official Account 成本

| 方案 | 月費 | 每月推播上限 |
|------|------|-------------|
| 免費版 | NT$0 | 200 則 |
| 輕用量版 | NT$800 | 3,000 則 |
| 中用量版 | NT$2,400 | 6,000 則 |

**MVP 驗證階段**：免費方案（200 則/月）足夠服務 50～100 個早期用戶（每人每日 1 則）。

#### 為什麼 LINE 是唯一正解

- LINE 台灣用戶 **2,200 萬**，滲透率 **90.7%**
- 農業部、各地農改場、農會都已用 LINE 群組服務農民
- 農民平均年齡 62 歲，LINE 是他們**唯一熟練使用的數位工具**
- 雲林縣已有「雲林農情通」LINE 服務成功案例

### 6.2 獲客漏斗

```
1. 免費 Web 儀表板（SEO + 農業論壇分享）
   ↓  轉化率 ~30%
2. 加 LINE Bot 好友（掃 QR Code）
   ↓  轉化率 ~50%
3. 免費試用 90 天
   ↓  轉化率 ~5～15%（關鍵轉化點）
4. 付費訂閱
```

### 6.3 具體獲客行動

| 行動 | 成本 | 預期效果 |
|------|------|----------|
| 在埔里農會公告欄張貼 QR Code | 接近 0 | 第一批 10～20 個用戶 |
| 參加埔里鎮農會農民大會 demo | 交通費 | 面對面取得信任 |
| 在 PTT 農業板 / 農民 FB 社團分享 | 0 | 線上曝光 |
| 台灣智慧農業展擺攤 | NT$5,000～20,000 | 接觸農會幹部和官員 |
| 聯繫農改場（台中、南投）| 0 | 學術背書 + 推薦管道 |

> [!tip] 口碑是一切
> 農民社群緊密，一個農民說「**這個有用，今天多賣了幾千元**」，立刻會有 5 個農民想用。**第一個成功案例比任何行銷預算都重要。**

## 八、風險與挑戰

### 8.1 風險矩陣

| 風險 | 影響 | 可能性 | 緩解策略 |
|------|------|--------|----------|
| 農民不願付費 | 🔴 高 | 🟡 中 | 先免費驗證價值，ROI 可視化 |
| 農糧署 API 停止/改版 | 🔴 高 | 🟢 低 | 政府開放資料政策穩定，備份爬取方案 |
| 被大型玩家複製 | 🟡 中 | 🟢 低 | 利基太小，大公司看不上 |
| 農民年齡層無法使用 | 🟡 中 | 🟡 中 | 只用 LINE，不做 APP |
| 市場價差不夠大（無價值） | 🔴 高 | 🟢 低 | 歷史數據顯示價差常態性存在 |
| 農會合作談判失敗 | 🟡 中 | 🟡 中 | 不依賴農會，B2C 也能存活 |

### 8.2 最大的不確定性

> **農民是否真的會因為「知道哪個市場比較好」而改變銷售行為？**

可能的阻礙：
- 農民與特定盤商有長期合作關係，不輕易換
- 送到較遠市場的運費可能吃掉價差
- 農民沒有足夠的議價能力（盤商不接受指定市場）

→ **這必須用 5～10 個農民的實際訪談來驗證，不能光靠數據推測。**

### 8.3 致命假設清單

在投入更多開發之前，以下假設必須驗證：

- [ ] **假設 1**：農民知道市場間存在顯著價差（不是理所當然的）
- [ ] **假設 2**：農民有能力選擇送貨到哪個市場（不是盤商決定）
- [ ] **假設 3**：價差扣除運費後仍有利可圖
- [ ] **假設 4**：農民願意使用 LINE Bot（不只是 LINE 聊天）
- [ ] **假設 5**：NT$200～300/月 是農民能接受的價格

## 九、行動藍圖

### Phase 0：假設驗證（第 1～4 週）🎯 最重要

> [!important] 在寫任何新代碼之前，先驗證商業假設

- [ ] 找 5 個埔里茭白筍農民面對面訪談
- [ ] 問題清單：
  - 你現在怎麼決定送哪個市場？
  - 你知道不同市場的價差嗎？
  - 如果有工具每天告訴你哪個市場最好，你會改變行為嗎？
  - 這個服務值多少錢？
- [ ] 驗證「假設 1～5」
- [ ] 產出：10 頁訪談筆記 + Go/No-Go 決定

### Phase 1：LINE Bot MVP（第 5～10 週）

- [ ] 替換 LINE Notify → LINE Messaging API
- [ ] 實作「每日行情摘要」推播（早上 6:00）
- [ ] 實作「價差警報」（當任一市場溢價超過 10%）
- [ ] 格式範例：

```
🟢 茭白筍行情速報 114.03.03

今日最佳市場：台北二（104）
  均價 NT$92/kg（+31.4%）

基準市場：台中市（400）
  均價 NT$70/kg

💡 送台北二，每噸多賺 NT$22,000
```

- [ ] 邀請 10～20 個農民免費試用

### Phase 2：早期變現（第 11～24 週）

- [ ] 加入「運費估算器」（整合距離 × 車趟成本）
- [ ] 付費功能上線：
  - 進階歷史分析（30/90/365 天）
  - 自訂警報閾值
  - 每週最佳市場報告
- [ ] 洽談埔里鎮農會合作
- [ ] 目標：**50 個免費用戶、5 個付費用戶**

### Phase 3：規模化（第 25～52 週）

- [ ] 農會白標版本上線
- [ ] 支援多種作物（埔里其他蔬菜）
- [ ] 考慮成立一人公司，申請政府補助
- [ ] 目標：**200 個用戶、30 個付費用戶、1～2 個農會客戶**

## 十、結論：可行性評估

### 評分卡

| 面向 | 分數（1-5） | 說明 |
|------|------------|------|
| 市場需求 | ⭐⭐⭐⭐ | 痛點真實，缺口明確，但需驗證行為改變意願 |
| 競爭壁壘 | ⭐⭐⭐ | 利基太小大公司不做，但技術無護城河 |
| 技術可行性 | ⭐⭐⭐⭐⭐ | MVP 已完成，加 LINE Bot 即可商業化 |
| 商業化難度 | ⭐⭐ | 農民付費意願是最大挑戰 |
| 獨立開發者適合度 | ⭐⭐⭐⭐ | 低成本、利基市場、技術驅動 |
| 收入天花板 | ⭐⭐ | 純 B2C 天花板低（年收 < NT$100 萬），需 B2B 突破 |

### 總體判斷

> **商業可行性：有條件的可行（Conditionally Viable）**

**值得做的理由**：
1. 市場缺口真實存在，你是目前唯一填補者
2. 技術已就位（MVP 完成），增量開發成本低
3. 運營成本極低（年 < NT$20,000），幾乎零風險
4. 地理集中度高（85% 在埔里），獲客效率高

**必須注意的風險**：
1. 收入天花板低 — 純茭白筍 × 純農民 ≈ 年收 NT$30～60 萬
2. 農民付費轉化率可能 < 5%
3. 需要 **Phase 0 的用戶訪談** 來確認商業假設

### 獨立開發者的最佳策略

```
                    不要這樣做
                    ┌──────────┐
                    │ 一開始就 │
                    │ 做付費功能│
                    └──────────┘
                         ✗

     應該這樣做
┌─────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│  訪談    │ →  │ 免費 LINE │ →  │  口碑擴散 │ →  │ 付費 +   │
│  5 個農民│    │ Bot 試用  │    │  50+ 用戶 │    │ 農會合作  │
└─────────┘    └──────────┘    └──────────┘    └──────────┘
  4 週           6 週            3 個月           6 個月
```

> [!quote] 最後提醒
> **不要急著寫付費功能。先花 4 週去埔里跟農民聊天。** 如果 5 個農民中有 3 個說「這個有用，我願意付 NT$200/月」，那就全力推進。如果大家都說「我不會因此改變送貨的市場」，那就需要調整方向（例如轉向服務盤商或農會，而非終端農民）。
